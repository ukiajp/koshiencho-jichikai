### 採用理由と廃止理由

**Netlify廃止**：無料枠の上限超過により全プロジェクト停止。追加費用なしで継続できないため廃止。

**Decap CMS廃止**：Netlify Identity（Netlify専用認証）に依存しており、ロリポップでは動作しない。また「GitHub web UI + AI補助」で同等の更新体験が実現できると判断。

**ロリポップ採用**：自治会で新規契約予定。FTPアクセス可能。月550円（スタンダードプラン）で静的サイトの運用に十分。

**GitHub web UI + Claude採用**：更新担当者（会長・副会長）はPC日常利用者。ブラウザ上でMarkdownを編集し、AIにファイル生成を依頼する運用が現実的と判断。GitHubアカウント引き継ぎは将来的にOrganizationアカウントへ移行予定。

## 6. イベントカレンダー 設計方針

### 2層構造

**【層1】定期活動 — ページに直接記載（更新不要）**

| 曜日・頻度 | 活動 | 時間 | 場所 |
|---|---|---|---|
| 毎週月・木 | グラウンドゴルフ | 9:00〜 | 月見里公園 |
| 毎週火 | いきいき体操 | 13:30〜 | 月見里会館 |
| 毎週水・木・金 | ダーツ | 13:00〜 | 月見里会館 |
| 月1回（第3月曜） | 花の会 | — | — |
| 月複数回 | ときわ会定例会・お誕生会 | — | — |
| 月2回（第2・4水曜） | ひばり会 | — | — |

**【層2】行事・イベント — `.md` ファイルで管理**

- 公園清掃・役員会・地蔵盆など不定期行事
- 甲子園球場日程（年1回シーズン前に一括登録）

### カレンダー表示

- **月別カレンダーグリッド**：当月のイベントを日付セルに表示、前月・翌月への切り替えボタン付き
- **カード一覧**：カレンダー下部に今後のイベントをカード形式で表示
- **詳細ページ**：カードクリックでサイト内詳細ページへ遷移

### 甲子園球場スケジュール

- 阪神タイガース公式戦・選抜・夏の甲子園をイベントとして登録
- 「急な変更は公式サイトをご確認ください」注釈を常設
- 他の自治会サイトにない差別化ポイント

## 9. 更新フロー（非エンジニア対応）

### 通常の更新手順

```
1. GitHub（ブラウザ）にログイン
2. src/content/ 配下の該当フォルダを開く
3. 「Add file」→「Create new file」を選択
4. Claudeに「この内容でイベントのmdファイルを作って」と頼んでコピペ
5. ファイル名を入力（例：20260401_公園清掃.md）
6. 「Commit changes」ボタンを押す
7. 約2〜3分後にサイトへ自動反映
```

### 写真のアップロード

```
1. GitHub の assets/images/ フォルダを開く
2. 「Add file」→「Upload files」
3. 画像をドラッグ＆ドロップ
4. 「Commit changes」ボタンを押す
5. mdファイルに画像パスを記載
```

## 10. デプロイ設計

```
GitHub push（web UIのCommitも含む）
      ↓
GitHub Actions 自動起動
      ↓
Eleventy ビルド（_site/ 生成）
      ↓
FTP でロリポップへ転送
      ↓
サイト反映（約2〜3分）
```

設定ファイル：`.github/workflows/deploy.yml`

## 11. デザイン方針

- モバイルファースト（スマホ表示最優先）
- カラー：深緑（#1a5c38）・白（#ffffff）・紺（#1e3a5f）
- 文字サイズ：16px以上（高齢者配慮）
- Tailwind CSS CDN使用（ビルド環境不要）

### トップページ 情報設計

住民が最も知りたい3つへの導線をトップに集約：
1. **次のイベント・行事**（カレンダーへのリンク）
2. **会館が空いているか**（Coubic予約へのCTA）
3. **最新のお知らせ**（3件表示）
